## 一、润色 Query（查询优化）

### 1.1 背景

当前问答流程：

```
用户提问 → [查询改写] → 向量检索 + BM25 → RRF 融合 → Reranker → LLM 生成
```

"查询改写"只在多轮对话时做指代消解（"它是什么？"→"RAG是什么？"）。单轮查询直接原文检索，缺少对用户表述质量的优化。

**痛点：**
- 用户口语化提问："请假要咋弄" → BM25 分词后关键词弱，向量语义也不准
- 拼写错误："公唣制度" → 向量和 BM25 都查不到
- 缺少同义扩展："报销流程" → 文档中写的是"费用报销申请流程"
- 关键词稀疏："公司的那个规定" → 几乎无检索价值

### 1.2 方案设计

在查询改写环节之后、检索之前，增加 **Query 润色** 步骤：

```
用户提问 → [查询改写·已有] → [Query 润色·新增] → 检索 → ...
```

#### 润色策略（三步走，LLM 一次调用完成）

| 步骤 | 说明 | 示例 |
|------|------|------|
| **拼写纠错** | 识别错别字并修正 | "公唣制度" → "公司制度" |
| **同义扩展** | 识别可能的同义表述，扩展查询 | "报销" → "报销 OR 费用申请 OR 报销流程" |
| **关键词提取** | 提取核心实体/概念，补充 BM25 检索 | "请假要咋弄" → 提取: 请假、申请、流程 |

#### 实现方式

**新增 Prompt（config/prompts.json）：**

```json
{
  "polish": {
    "name": "查询润色 Prompt",
    "description": "对用户查询进行纠错、扩展、关键词提取",
    "system": "你是一个查询优化助手。对用户输入的查询进行优化处理。\n\n输出严格 JSON 格式：\n{\"corrected\": \"纠错后的查询\", \"expanded\": \"扩展后的查询（包含同义表述，用空格连接）\", \"keywords\": [\"关键词1\", \"关键词2\"]}\n\n规则：\n1. corrected: 修正明显的错别字和语法错误，保持原意\n2. expanded: 在 corrected 基础上，补充可能的同义表述/近义词，用于提升召回率。不超过原查询的 2 倍长度\n3. keywords: 提取 3-5 个核心关键词/实体，用于 BM25 检索\n4. 如果查询本身质量高（无错别字、表述清晰），corrected 可与原文相同\n5. 不要编造查询中不存在的语义",
    "user": "用户查询：{question}\n\n请输出优化结果 JSON："
  }
}
```

**新增函数（app/core/llm.py）：**

```python
def polish_query(question: str) -> dict:
    """
    润色查询：拼写纠错 + 同义扩展 + 关键词提取。
    返回 {"corrected": str, "expanded": str, "keywords": list[str]}
    失败时返回原始问题，不影响正常流程。
    """
```

**修改检索流程（app/api/query_routes.py）：**

```python
# 在 search_question 确定之后、检索之前

# 1) 润色
polished = polish_query(search_question)
corrected = polished.get("corrected", search_question)
expanded = polished.get("expanded", "")
keywords = polished.get("keywords", [])

# 2) 向量检索用 expanded（更丰富的语义）
vector_results = vector_query(expanded or corrected, ...)

# 3) BM25 检索用 expanded + keywords 拼接（更多关键词）
bm25_query = expanded + " " + " ".join(keywords)
```

#### 检索策略调整

| 检索通道 | 查询输入 | 原因 |
|----------|----------|------|
| 向量检索 | `expanded`（扩展查询） | 同义词提升语义召回 |
| BM25 | `expanded + keywords` | 关键词越多，精确匹配越准 |
| 最终回答 | 原始 `question` | 回答要针对用户原始意图 |

#### 前端改动

问答页增加润色可视化（可选，不阻塞核心流程）：
- 检索结果区域顶部显示："已优化查询：xxx"（灰色小字）
- 用户可点击展开看到纠错/扩展/关键词详情

#### 开关与降级

- 新增系统配置项 `query_polish.enabled`（默认开启）
- LLM 调用失败 → 降级到原始查询，不报错
- 超时保护：润色 LLM 调用 timeout=3s，超时跳过
- 缓存：润色结果随查询结果一起缓存，不额外缓存

## 二、更多文档格式支持

### 2.1 背景

当前支持的格式：

| 格式 | 加载方式 | 状态 |
|------|----------|------|
| `.pdf` | OCR（PaddleOCR）→ Markdown | ✅ 已支持 |
| `.docx` | python-docx 提取文本 | ✅ 已支持 |
| `.txt` | 直接读取 | ✅ 已支持 |
| `.md` | 直接读取 | ✅ 已支持 |

**缺失但高频需求的格式：**

| 格式 | 典型场景 |
|------|----------|
| **Excel (.xlsx/.xls)** | 报表、台账、数据表、人员名单 |
| **CSV** | 导出数据、批量数据 |
| **PowerPoint (.pptx)** | 汇报材料、培训文档 |

### 2.2 方案设计

#### Excel (.xlsx / .xls)

**加载策略：**
- 用 `openpyxl` 读取 `.xlsx`，用 `xlrd` 读取 `.xls`（兼容老格式）
- 每个工作表 → 独立文本段落
- 表头行作为列名，每行数据转为 "列名: 值" 的描述性文本
- 超过 1000 行的表格分段处理（避免单块过大）

**转换示例：**

原始 Excel：
```
| 姓名   | 部门   | 入职日期   |
|--------|--------|------------|
| 张三   | 研发部 | 2024-03-01 |
| 李四   | 产品部 | 2024-06-15 |
```

转换后文本：
```
## Sheet1（员工表）

第 1 条记录：姓名为张三，部门为研发部，入职日期为 2024-03-01
第 2 条记录：姓名为李四，部门为产品部，入职日期为 2024-06-15
```

**为什么转描述性文本而非保留表格格式？**
- 向量检索基于语义，"姓名为张三"比原始表格数据更容易被 Embedding 理解
- BM25 分词也更友好

#### CSV

**加载策略：**
- 用 Python 内置 `csv` 模块读取
- 自动检测编码（utf-8 → gbk → gb2312 fallback）
- 转换逻辑同 Excel：表头 + 描述性文本

#### PowerPoint (.pptx)

**加载策略：**
- 用 `python-pptx` 读取
- 每张幻灯片 → 一个文本段落
- 提取内容：标题 + 正文文本框 + 备注页
- 忽略图片（无法提取文字内容，除非后续接入多模态）
- 跳过空白幻灯片

**转换示例：**

原始 PPT 幻灯片：
```
Slide 1:
  标题: 2024年度总结
  正文: 完成了XX项目上线...
  备注: 需要补充数据
```

转换后文本：
```
## 幻灯片 1

标题：2024年度总结
内容：完成了XX项目上线...
备注：需要补充数据
```

### 2.3 依赖变更

**新增 pip 依赖（requirements.txt）：**

```
openpyxl==3.1.5       # .xlsx 读取
xlrd==2.0.1           # .xls 读取（兼容老格式）
python-pptx==1.0.2    # .pptx 读取
chardet==5.2.0        # CSV 编码检测
```

> CSV 用内置 `csv` 模块，编码检测用 `chardet`。

### 2.4 代码改动清单

| 文件 | 改动 |
|------|------|
| `app/core/loader.py` | 新增 `_load_excel()`、`_load_csv()`、`_load_pptx()`，扩展 `loaders` 字典 |
| `app/api/doc_routes.py` | `ALLOWED_EXTENSIONS` 增加 `.xlsx`, `.xls`, `.csv`, `.pptx` |
| `app/static/js/pages/upload.js` | 文件选择器 `accept` 属性更新 |
| `requirements.txt` | 新增 4 个依赖 |

### 2.5 特殊处理

| 场景 | 处理方式 |
|------|----------|
| Excel 多 Sheet | 每个 Sheet 独立段落，以 `## Sheet名` 分隔 |
| Excel 超大文件 (>10MB) | 同步处理即可（非 OCR，速度较快），超 50MB 仍拒绝 |
| CSV 编码问题 | chardet 检测 → utf-8 → gbk → gb2312 逐个尝试 |
| PPT 含图片 | 跳过，日志警告"幻灯片 X 包含未提取的图片" |
| .xls 老格式 | xlrd 兼容，与 .xlsx 统一输出格式 |
